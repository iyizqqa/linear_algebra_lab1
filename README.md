# Лабораторная работа №1 — методы решения СЛАУ

В этой лабораторной сравниваются несколько способов решения системы линейных уравнений:

```math
Ax=b
```

где `A` — квадратная матрица, `b` — правая часть, `x` — искомый вектор.

В программе реализованы:

- метод Гаусса без выбора ведущего элемента;
- метод Гаусса с выбором главного элемента по столбцу;
- LU-разложение;
- прямая и обратная подстановки;
- подсчёт невязки и относительной ошибки;
- эксперименты на случайных матрицах и на матрицах Гильберта.

Основная цель — сравнить методы по времени и точности, а также проверить, насколько LU-разложение удобно, если нужно решать много систем с одной и той же матрицей, но с разными правыми частями.

## Используемые методы

### Метод Гаусса без выбора ведущего элемента

Метод Гаусса приводит систему

```math
Ax=b
```

к верхнетреугольному виду:

```math
Ux=\tilde b
```

На каждом шаге зануляются элементы ниже главной диагонали. Для этого считается коэффициент

```math
m_{ik}=\frac{a_{ik}}{a_{kk}}
```

и из строки `i` вычитается строка `k`, умноженная на этот коэффициент:

```math
a_{ij} \leftarrow a_{ij} - m_{ik}a_{kj}
```

```math
b_i \leftarrow b_i - m_{ik}b_k
```

После прямого хода остаётся решить верхнетреугольную систему. Это делается обратной подстановкой:

```math
x_i = \frac{b_i - \sum\limits_{j=i+1}^{n-1} u_{ij}x_j}{u_{ii}}
```

Минус этого варианта в том, что он никак не защищён от маленького диагонального элемента. Если `a[k][k]` близок к нулю, погрешность может сильно вырасти.

---

### Метод Гаусса с выбором главного элемента

Этот вариант работает почти так же, но перед исключением выбирается строка с максимальным по модулю элементом в текущем столбце:

```math
p = \arg\max_{i \ge k} |a_{ik}|
```

Потом строки `k` и `p` меняются местами.

Это простое изменение делает метод устойчивее, потому что уменьшается шанс деления на очень маленькое число. Поэтому на практике вариант с выбором главного элемента обычно надёжнее обычного Гаусса.

---

### LU-разложение

LU-разложение представляет матрицу `A` как произведение двух треугольных матриц:

```math
A=LU
```

где:

- `L` — нижнетреугольная матрица;
- `U` — верхнетреугольная матрица;
- на диагонали `L` стоят единицы.

В программе используется вариант без перестановок строк. Для элементов `U`:

```math
u_{ij}=a_{ij}-\sum\limits_{k=0}^{i-1}l_{ik}u_{kj}, \quad j \ge i
```

Для элементов `L`:

```math
l_{ji}=\frac{a_{ji}-\sum\limits_{k=0}^{i-1}l_{jk}u_{ki}}{u_{ii}}, \quad j>i
```

После разложения исходная система переписывается так:

```math
LUx=b
```

Дальше она решается в два этапа.

Сначала решается система:

```math
Ly=b
```

Это прямая подстановка.

Потом решается:

```math
Ux=y
```

Это обратная подстановка.

Главный плюс LU-разложения в том, что матрицу можно разложить один раз, а потом использовать это разложение для разных правых частей:

```math
Ax_1=b_1, \quad Ax_2=b_2, \quad \ldots, \quad Ax_k=b_k
```

То есть дорогой этап разложения выполняется один раз, а дальше остаются только быстрые подстановки.

---

### Невязка

Чтобы проверить, насколько хорошо найденное решение подходит к исходной системе, считается невязка:

```math
r=Ax-b
```

В программе используется евклидова норма:

```math
\|r\|_2 = \sqrt{\sum\limits_i r_i^2}
```

Чем меньше невязка, тем ближе найденное решение к выполнению исходной системы.

---

### Относительная погрешность

Для матрицы Гильберта точное решение задаётся заранее:

```math
x=(1,1,\ldots,1)^T
```

Поэтому можно сравнить найденное решение `\tilde x` с точным:

```math
\frac{\|\tilde x-x\|_2}{\|x\|_2}
```

Эта величина показывает уже не просто качество подстановки решения в систему, а реальную ошибку найденного вектора.

---

### Матрица Гильберта

Матрица Гильберта строится так:

```math
H_{ij}=\frac{1}{i+j+1}
```

В формуле используется индексация с нуля.

Такие матрицы плохо обусловлены. Это значит, что даже маленькие ошибки округления могут сильно портить ответ. Поэтому матрица Гильберта хорошо подходит для проверки устойчивости методов.

Для наглядности можно посчитать ускорение LU относительно метода Гаусса:

| k | Ускорение LU |

| 1 | 0.93× |

При одной правой части LU немного медленнее, потому что разложение само по себе занимает время. Но уже при `k = 10` LU становится быстрее примерно в `8.72` раза, а при `k = 100` — примерно в `63.60` раза.

Причина в том, что метод Гаусса каждый раз заново решает систему почти с нуля. LU один раз раскладывает матрицу, а дальше для каждой новой правой части выполняет только прямую и обратную подстановки.

Для `n = 15` методы не смогли корректно завершить вычисления. В коде стоит проверка на слишком маленький ведущий элемент:

```math
EPS=10^{-12}
```

Если pivot меньше этого значения, метод считает систему проблемной и возвращает ошибку. Для матрицы Гильберта размера 15 это как раз и происходит.

## Вывод

В работе были реализованы три способа решения СЛАУ: обычный метод Гаусса, метод Гаусса с выбором главного элемента и LU-разложение.

По результатам экспериментов получилось следующее:

1. Для решения одной системы метод Гаусса с выбором главного элемента оказался самым практичным вариантом: он достаточно быстрый и даёт хорошую точность.
2. Выбор главного элемента заметно улучшает устойчивость. На больших случайных матрицах невязка у метода с выбором намного меньше, чем у метода без выбора.
3. LU-разложение особенно полезно, когда правая часть меняется, а матрица остаётся той же. При `k = 100` оно оказалось быстрее метода Гаусса примерно в `63.60` раза.
4. Матрица Гильберта показала, что плохо обусловленные системы могут давать большую ошибку даже при маленькой невязке.
5. Если нужно решить одну систему, лучше брать Гаусс с выбором главного элемента. Если систем много и матрица одна и та же, лучше использовать LU-разложение.
